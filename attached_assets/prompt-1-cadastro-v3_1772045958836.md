# PROMPT REPLIT — Adicionar Métricas de Rating ao Cadastro de Restaurante Ativo

## Contexto

O sistema Mesa ADS possui a tabela `active_restaurants` com restaurantes parceiros. Este prompt adiciona campos ao formulário existente (`/restaurantes/novo` e `/restaurantes/:id`) para alimentar um sistema de rating interno que classifica restaurantes em tiers (Bronze, Prata, Ouro, Diamante) com multiplicadores de preço.

---

## Campos que JÁ EXISTEM na tabela `active_restaurants` e serão REUTILIZADOS

Estes campos já estão no schema. **NÃO duplicar.** Serão lidos pela função de rating:

| Campo existente | Tipo atual | Uso no rating |
|---|---|---|
| `monthlyCustomers` | integer | Fluxo mensal — peso 25% |
| `tableCount` | integer | Número de mesas — peso 10% |
| `instagram` | varchar | Já existe, não duplicar |

Outros campos existentes que NÃO entram no cálculo do rating mas são dados complementares úteis:
- `seatCount`, `busyDays`, `busyHours`, `socialClass`, `monthlyDrinksSold`

---

## Novos campos a adicionar na tabela `active_restaurants`

Seguir o padrão camelCase já usado no schema (ex: `tableCount`, `monthlyCustomers`, `contactName`):

```sql
-- Campos de input (preenchidos pelo usuário)
ALTER TABLE active_restaurants ADD COLUMN "ticketMedio"       numeric      NOT NULL DEFAULT 0;
ALTER TABLE active_restaurants ADD COLUMN "avgStayMinutes"    integer      NOT NULL DEFAULT 0;
ALTER TABLE active_restaurants ADD COLUMN "locationRating"    integer      NOT NULL DEFAULT 1;
ALTER TABLE active_restaurants ADD COLUMN "venueType"         integer      NOT NULL DEFAULT 1;
ALTER TABLE active_restaurants ADD COLUMN "digitalPresence"   integer      NOT NULL DEFAULT 1;
ALTER TABLE active_restaurants ADD COLUMN "primaryDrink"      varchar;

-- Campos calculados (preenchidos automaticamente pelo sistema)
ALTER TABLE active_restaurants ADD COLUMN "ratingScore"       numeric(3,2);
ALTER TABLE active_restaurants ADD COLUMN "ratingTier"        varchar;
ALTER TABLE active_restaurants ADD COLUMN "ratingMultiplier"  numeric(3,2);
ALTER TABLE active_restaurants ADD COLUMN "ratingUpdatedAt"   timestamp;
```

**Constraints:**
- `ticketMedio` >= 0
- `avgStayMinutes` >= 0
- `locationRating` BETWEEN 1 AND 5
- `venueType` BETWEEN 1 AND 5
- `digitalPresence` BETWEEN 1 AND 5

## Alterações no formulário (`/restaurantes/novo` e `/restaurantes/:id`)

Adicionar uma nova seção **"Perfil Publicitário"** após os campos operacionais (tableCount, seatCount, monthlyCustomers) e antes de excludedCategories/notes.

### Campo 1: `ticketMedio`
- **Input:** number, formato moeda BRL (R$)
- **Label:** "Ticket médio por cliente (R$)"
- **Placeholder:** "Ex: 65.00"
- **Validação:** valor positivo, mínimo 5.00, obrigatório
- **Tooltip:** "Valor médio gasto por cliente em uma visita."

### Campo 2: `avgStayMinutes`
- **Input:** number (inteiro, em minutos)
- **Label:** "Tempo médio de permanência (minutos)"
- **Placeholder:** "Ex: 45"
- **Validação:** inteiro positivo, mínimo 10, máximo 300, obrigatório
- **Tooltip:** "Tempo médio que o cliente permanece sentado no estabelecimento."

### Campo 3: `locationRating`
- **Input:** select dropdown
- **Label:** "Perfil da localização"
- **Opções:**
  - `1` → "Periferia / Baixo fluxo de pedestres"
  - `2` → "Bairro residencial de classe média"
  - `3` → "Zona comercial ativa / Avenida movimentada"
  - `4` → "Bairro nobre / Polo gastronômico"
  - `5` → "Ponto turístico / Localização premium (orla, centro histórico)"
- **Validação:** obrigatório

### Campo 4: `venueType`
- **Input:** select dropdown
- **Label:** "Tipo do estabelecimento"
- **Opções:**
  - `1` → "Lanchonete / Fast food"
  - `2` → "Bar popular / Boteco"
  - `3` → "Restaurante casual"
  - `4` → "Gastropub / Bar temático / Cervejaria artesanal"
  - `5` → "Fine dining / Rooftop / Lounge"
- **Validação:** obrigatório

### Campo 5: `digitalPresence`
- **Input:** select dropdown
- **Label:** "Presença digital e redes sociais"
- **Opções:**
  - `1` → "Sem redes sociais ou inativo"
  - `2` → "Redes sociais com pouca atividade"
  - `3` → "Ativo nas redes, até 5.000 seguidores"
  - `4` → "Engajado, 5.000 a 20.000 seguidores"
  - `5` → "Influente, 20.000+ seguidores ou perfil verificado"
- **Validação:** obrigatório

### Campo 6: `primaryDrink` (opcional)
- **Input:** select dropdown
- **Label:** "Tipo de bebida predominante"
- **Opções:**
  - `"cerveja"` → "Cerveja / Chopp"
  - `"drinks"` → "Drinks / Coquetéis"
  - `"vinho"` → "Vinho"
  - `"nao_alcoolico"` → "Sucos / Não alcoólicos"
  - `"variado"` → "Variado / Sem predominância"
- **Validação:** opcional (pode ser null)

## Alterações no Perfil do Restaurante (`/restaurantes/perfil/:id`)

Adicionar duas seções na página de perfil:

### Seção "Perfil Publicitário"
Exibir os campos novos em modo leitura:
- Ticket médio formatado como moeda: "R$ 65,00"
- Tempo de permanência: "45 min"
- Localização, tipo, presença digital: exibir o label da opção selecionada
- Bebida predominante: exibir o label ou "—" se null

### Card de Rating
- Se `ratingScore` não for null: exibir score (X.XX / 5.00), badge de tier com cor, multiplicador (X.X×)
- Se `ratingScore` for null: exibir "Classificação pendente — preencha os campos de perfil publicitário"
- Cores dos tiers: Bronze (#CD7F32), Prata (#C0C0C0), Ouro (#FFD700), Diamante (#4169E1)

---

## Alterações na listagem (`/restaurantes`)

Na tabela existente, adicionar coluna **"Rating"**:
- Conteúdo: badge colorido com tier + score. Ex: badge dourado "OURO 3.70"
- Se null: mostrar "—"
- Permitir **ordenação** por `ratingScore` (nulls por último)
- Permitir **filtro** por tier (checkboxes: Bronze, Prata, Ouro, Diamante)

---

## UX e padrões

1. Seguir o mesmo padrão visual das seções existentes do formulário (mesmos componentes, mesmo estilo)
2. Tooltips com o mesmo componente já usado no sistema
3. Mobile-friendly (equipe comercial cadastra durante visita)
4. Campos `ratingScore`, `ratingTier`, `ratingMultiplier`, `ratingUpdatedAt` NÃO têm input no formulário — são somente leitura
5. Se o restaurante vier da prospecção (`parentRestaurantId` preenchido), os novos campos começam nos defaults e são preenchidos na primeira edição

## O que NÃO fazer

- NÃO implementar a lógica de cálculo do rating (Prompt 2)
- NÃO alterar o Simulador, Campanhas, Prospecção ou outras páginas
- NÃO criar novas tabelas no banco
- NÃO duplicar campos que já existem (`monthlyCustomers`, `tableCount`, `instagram`)
- NÃO alterar o schema de nenhuma outra tabela
