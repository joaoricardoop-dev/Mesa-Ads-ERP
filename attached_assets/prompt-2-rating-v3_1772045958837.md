# PROMPT REPLIT — Calculadora Interna de Rating do Restaurante

## Contexto

A tabela `active_restaurants` já possui campos operacionais e de perfil publicitário (adicionados no prompt anterior). Este prompt implementa o **cálculo automático do rating** que classifica cada restaurante em tiers e gera um multiplicador de preço.

---

## Mapeamento exato dos campos para o cálculo

Os 7 campos usados no rating vêm todos da tabela `active_restaurants`:

| Variável | Campo no banco | Tipo | Origem | Peso |
|---|---|---|---|---|
| Fluxo mensal | `monthlyCustomers` | integer | Já existia | 25% |
| Ticket médio | `ticketMedio` | numeric | Prompt 1 | 20% |
| Permanência | `avgStayMinutes` | integer | Prompt 1 | 15% |
| Localização | `locationRating` | integer (1-5) | Prompt 1 | 15% |
| Mesas | `tableCount` | integer | Já existia | 10% |
| Perfil | `venueType` | integer (1-5) | Prompt 1 | 10% |
| Digital | `digitalPresence` | integer (1-5) | Prompt 1 | 5% |

Campos calculados salvos na mesma tabela:

| Campo | Tipo | Descrição |
|---|---|---|
| `ratingScore` | numeric(3,2) | Score de 1.00 a 5.00 |
| `ratingTier` | varchar | "bronze", "prata", "ouro" ou "diamante" |
| `ratingMultiplier` | numeric(3,2) | 1.00, 1.40, 1.80 ou 2.30 |
| `ratingUpdatedAt` | timestamp | Data/hora do último cálculo |

## Função de cálculo

Criar função **compartilhável** entre frontend (preview) e backend (oficial). Usar os nomes exatos dos campos do banco:

```javascript
import { RATING_CONFIG } from './rating-config'

function converterParaPontos(valor, faixas) {
  for (const faixa of faixas) {
    if (valor <= faixa.max) return faixa.pontos
  }
  return 5
}

/**
 * Verifica se o restaurante tem todos os campos necessários para calcular o rating.
 * Usa os nomes exatos da tabela active_restaurants.
 */
export function temCamposRatingCompletos(restaurante) {
  return (
    restaurante.monthlyCustomers != null && restaurante.monthlyCustomers > 0 &&
    restaurante.tableCount != null && restaurante.tableCount > 0 &&
    restaurante.ticketMedio != null && restaurante.ticketMedio > 0 &&
    restaurante.avgStayMinutes != null && restaurante.avgStayMinutes > 0 &&
    restaurante.locationRating != null && restaurante.locationRating >= 1 &&
    restaurante.venueType != null && restaurante.venueType >= 1 &&
    restaurante.digitalPresence != null && restaurante.digitalPresence >= 1
  )
}

/**
 * Calcula o rating completo de um restaurante.
 * @param restaurante - objeto com os campos da tabela active_restaurants
 */
export function calcularRating(restaurante) {
  const { pesos, faixas, tiers } = RATING_CONFIG

  // 1. Converter métricas brutas em pontos (1-5)
  const pontos = {
    fluxo:       converterParaPontos(restaurante.monthlyCustomers, faixas.fluxo),
    ticket:      converterParaPontos(restaurante.ticketMedio, faixas.ticket),
    permanencia: converterParaPontos(restaurante.avgStayMinutes, faixas.permanencia),
    localizacao: restaurante.locationRating,      // já é 1-5
    mesas:       converterParaPontos(restaurante.tableCount, faixas.mesas),
    perfil:      restaurante.venueType,            // já é 1-5
    digital:     restaurante.digitalPresence,      // já é 1-5
  }

  // 2. Score ponderado (arredondado para 2 casas)
  const score = Math.round((
    (pontos.fluxo * pesos.fluxo) +
    (pontos.ticket * pesos.ticket) +
    (pontos.permanencia * pesos.permanencia) +
    (pontos.localizacao * pesos.localizacao) +
    (pontos.mesas * pesos.mesas) +
    (pontos.perfil * pesos.perfil) +
    (pontos.digital * pesos.digital)
  ) * 100) / 100

  // 3. Determinar tier
  const tierInfo = tiers.find(t => score <= t.maxScore) || tiers[tiers.length - 1]

  return {
    score,                                  // ex: 3.70
    tier: tierInfo.nome,                    // ex: "ouro"
    multiplicador: tierInfo.multiplicador,  // ex: 1.80
    cor: tierInfo.cor,                      // ex: "#FFD700"
    detalhamento: {
      fluxo:       { valor: restaurante.monthlyCustomers,  pontos: pontos.fluxo,       peso: pesos.fluxo },
      ticket:      { valor: restaurante.ticketMedio,       pontos: pontos.ticket,      peso: pesos.ticket },
      permanencia: { valor: restaurante.avgStayMinutes,    pontos: pontos.permanencia, peso: pesos.permanencia },
      localizacao: { valor: restaurante.locationRating,    pontos: pontos.localizacao, peso: pesos.localizacao },
      mesas:       { valor: restaurante.tableCount,        pontos: pontos.mesas,       peso: pesos.mesas },
      perfil:      { valor: restaurante.venueType,         pontos: pontos.perfil,      peso: pesos.perfil },
      digital:     { valor: restaurante.digitalPresence,   pontos: pontos.digital,     peso: pesos.digital },
    }
  }
}
```

## Triggers de cálculo automático

### Trigger 1 — Ao CRIAR restaurante (INSERT em `active_restaurants`)

No handler de criação de restaurante ativo, após o INSERT:

```javascript
async function criarRestauranteAtivo(dados) {
  const restaurante = await db.activeRestaurant.create(dados)

  if (temCamposRatingCompletos(restaurante)) {
    const rating = calcularRating(restaurante)
    await db.activeRestaurant.update(restaurante.id, {
      ratingScore: rating.score,
      ratingTier: rating.tier,
      ratingMultiplier: rating.multiplicador,
      ratingUpdatedAt: new Date()
    })
  }

  return restaurante
}
```

### Trigger 2 — Ao EDITAR restaurante (UPDATE em `active_restaurants`)

Recalcular quando qualquer dos 7 campos de rating mudar:

```javascript
const CAMPOS_RATING = [
  'monthlyCustomers', 'tableCount',           // já existiam
  'ticketMedio', 'avgStayMinutes',            // novos
  'locationRating', 'venueType', 'digitalPresence'  // novos
]

async function atualizarRestauranteAtivo(id, dadosAlterados) {
  const restaurante = await db.activeRestaurant.update(id, dadosAlterados)

  const alterouRating = CAMPOS_RATING.some(campo => campo in dadosAlterados)

  if (alterouRating) {
    const dadosCompletos = await db.activeRestaurant.findById(id)
    if (temCamposRatingCompletos(dadosCompletos)) {
      const rating = calcularRating(dadosCompletos)
      await db.activeRestaurant.update(id, {
        ratingScore: rating.score,
        ratingTier: rating.tier,
        ratingMultiplier: rating.multiplicador,
        ratingUpdatedAt: new Date()
      })
    } else {
      // Se algum campo ficou incompleto, limpar o rating
      await db.activeRestaurant.update(id, {
        ratingScore: null,
        ratingTier: null,
        ratingMultiplier: null,
        ratingUpdatedAt: null
      })
    }
  }

  return restaurante
}
```

**IMPORTANTE:** Se o formulário existente já faz o update via uma rota genérica (ex: `PUT /api/active-restaurants/:id`), inserir a lógica de recálculo NESSA rota existente. Não criar rota nova para isso.

```javascript
async function recalcularTodosRatings() {
  const restaurantes = await db.activeRestaurant.findAll()
  let atualizados = 0, ignorados = 0

  for (const r of restaurantes) {
    if (!temCamposRatingCompletos(r)) {
      ignorados++
      continue
    }
    const rating = calcularRating(r)
    await db.activeRestaurant.update(r.id, {
      ratingScore: rating.score,
      ratingTier: rating.tier,
      ratingMultiplier: rating.multiplicador,
      ratingUpdatedAt: new Date()
    })
    atualizados++
  }

  return { total: restaurantes.length, atualizados, ignorados }
}
```

## Edge cases

1. **Restaurante com campos novos = 0 (defaults):** `ticketMedio = 0`, `avgStayMinutes = 0`, `locationRating = 1`, `venueType = 1`, `digitalPresence = 1` — a validação `temCamposRatingCompletos` rejeita valores 0 para ticket e permanência. Rating fica null até edição.
2. **Restaurante migrado da prospecção:** `parentRestaurantId` preenchido. Campos novos ficam nos defaults. Rating null até primeira edição.
3. **Score nos limites:** 2.00 = Bronze. 2.01 = Prata. Tiers usam `<=` no limite superior.
4. **Update que remove valor:** Se um campo de rating for setado como null ou 0, limpar o rating inteiro (score, tier, multiplicador = null).
5. **Permissões:** Verificar `role` do usuário na tabela `users`. Apenas `admin` pode usar recálculo em massa. `admin` e `user` podem editar restaurantes. `viewer` (se existir) não pode editar.
